# 🚨 [Bug Report] Ghost Input 및 무한 루프 버그 발생

## 1. 버그 증상 (Symptoms)
* **현상:** 윈도우 파일 탐색기에서 마우스 휠이나 스크롤바를 내려 파일을 찾으려고 하면, 1초에도 수십 번씩 맨 위로 강제로 튕겨 올라가는(스크롤되는) 심각한 오작동이 발생함.
* **추가 현상:** 파일 다중 선택 꼬임, 타이핑 시 글자 덮어쓰기 등 기괴한 현상 동반.
* **입력 모니터링 결과:** 사용자가 아무 키도 누르지 않은 상태에서 백그라운드로 `Key.shift`, `Key.home`, `Key.ctrl_l` 조합이 초당 수십 회씩 폭탄처럼 무한히 전송되고 있는 것을 확인.

## 2. 원인 분석 (Root Cause)
* 위 모니터링 된 키보드 입력은 **'자동영타세팅' 프로그램(`auto_eng_gui.pyw`)의 `on_typo_hotkey()` 함수에서 텍스트를 드래그하고 복사하기 위해 사용하는 단축키(`Shift + Home`, `Ctrl + C`)**와 완전히 일치함.
* **무한 루프 발생:** 매크로가 텍스트를 복사하기 위해 자체적으로 발생시킨 가상 키(Simulated Keypress) 이벤트(예: `VK_SHIFT` 누름)를, 이 프로그램의 키보드 훅(Keyboard Hook)이나 단축키 리스너가 "사용자가 트리거 단축키를 눌렀다"고 다시 오인식하는 것으로 강하게 추정됨.
* 비록 `on_typo_hotkey()` 내부에 `_typo_running` 플래그로 중복 실행을 막고 있더라도, 가상 키보드 신호가 이벤트 큐에 쌓이거나, `trigger_mod_count` (예: Shift 연속 누름) 조건을 충족시켜버리면서 플래그 해제 직후 곧바로 다시 함수가 호출되는 데드락/무한 루프에 빠짐.

## 3. 해결 및 수정 가이드라인 (Recommendations for the Agent)
이 문서를 읽고 있는 안티그래비티는 다음 사항들을 점검하여 코드를 수정해야 함.

1. **가상 입력(Artificial Input) 무시 처리:**
   * `keyboard` 라이브러리의 `hook`을 사용할 때, 프로그램 자신이 발생시킨 매크로 입력은 감지하지 않도록 처리해야 함. 
   * `keyboard.KeyboardEvent` 객체에는 이 구분을 위한 속성이 있을 수 있으나, 가장 확실한 방법은 단축키 트리거 로직에서 플래그 상태를 철저히 검사하는 것임.
2. **트리거 단축키 횟수 카운터 초기화 로직 점검:**
   * Modifier 연속 누름(예: Shift 2번)을 감지하는 부분에서, `on_typo_hotkey()` 가 실행된 후에는 `mod_key_count` 같은 상태 변수가 완전히 0으로 초기화되고, 매크로가 끝난 직후의 Modifier 상태를 1회차 누름으로 오인하지 않도록 `time.sleep()` 쿨다운이나 타임스탬프를 엄격하게 적용해야 함.
3. **`si_hotkey()` 및 `SendInput` 실행 중 Hook 비활성화:**
   * 매크로가 `Shift+Home`, `Ctrl+C` 등을 쏘는 동안에는 아예 단축키 감지 훅을 일시 중지(unhook)하거나 전역 변수로 입력 이벤트를 무조건 `return` 하도록 방어 코드를 짤 것.

이 버그로 인해 컴퓨터 사용이 불가능할 정도의 심각한 시스템 간섭이 발생하므로, 최우선적으로 핫픽스(Hotfix)가 필요함!
